## Usage

The Tooltip component provides contextual information when users hover or focus on elements. It's built with React Aria for excellent accessibility support.

```tsx
import { Tooltip, TooltipTrigger } from "@/registry/ui/tooltip"
import { Button } from "@/registry/ui/button"

<TooltipTrigger>
  <Button>Hover me</Button>
  <Tooltip>This is a helpful tooltips</Tooltip>
</TooltipTrigger>
```

## Accessibility

- **Keyboard Navigation**: Tooltips appear on focus and disappear when focus moves away
- **Screen Readers**: Tooltip content is announced when the trigger receives focus
- **ARIA Attributes**: Proper `aria-describedby` relationship between trigger and tooltip
- **Focus Management**: Tooltips don't trap focus and work with keyboard navigation
- **Timing**: Respects user preferences for reduced motion and timing

## Best Practices

- Keep tooltip content concise and helpful
- Don't put interactive elements inside tooltips
- Use tooltips for supplementary information, not essential content
- Consider the tooltip placement to avoid covering important UI elements
- Test with keyboard navigation and screen readers

## Advanced Usage

### Custom Delays

```tsx
<TooltipTrigger delay={300} closeDelay={100}>
  <Button>Quick tooltip</Button>
  <Tooltip>Appears faster, stays longer</Tooltip>
</TooltipTrigger>
```

### Without Arrow

```tsx
<TooltipTrigger>
  <Button>Clean tooltip</Button>
  <Tooltip showArrow={false}>Clean tooltip without arrow</Tooltip>
</TooltipTrigger>
```
